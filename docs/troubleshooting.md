# トラブルシューティングマニュアル

本スクリプトの実行中に発生しやすいエラーとその対処法のまとめです。不具合が起きた場合は、まずこのドキュメントを確認してください。

---

## 1. ffmpeg / ffprobe 関連のエラー

### 🚨 症状：「システムに 'ffmpeg' コマンドが見つかりません」と表示される
- **原因**: `ffmpeg` がパソコンにインストールされていないか、インストール直後で環境変数が反映されていません。
- **対処法**:
  1. ターミナル（PowerShell）を開き、`winget install Gyan.FFmpeg` を実行してインストールします。
  2. インストールが既に済んでいる場合は、**一度ターミナル（またはVS Codeなどのエディタ）を完全に閉じて、新しく開き直して**から再度実行してください。

### 🚨 症状：動画ファイルを指定した際、音声の抽出（m4a生成）で失敗する
- **原因**: 動画内の音声フォーマットが特殊であるか、動画ファイル自体が破損している可能性があります。
- **対処法**:
  - スクリプト側で自動的に汎用的な「AAC形式」への再エンコード抽出（第2作戦）に切り替わりますが、それでも失敗する場合は、一度フリーソフト等で動画を通常の `.mp4` に再変換してから試してください。

---

## 2. メモリ不足（MemoryError）関連のエラー

### 🚨 症状：モデルの読み込み中にプログラムが強制終了する
- **原因**: 使用しているパソコンのメモリ（RAM）またはグラフィックボードのメモリ（VRAM）が不足しています。特に `medium` や `large` モデルを指定したときに発生しやすいです。
- **対処法**:
  - `src/main.py` の上部にある【初期設定エリア】の `DEFAULT_MODEL_SIZE` を、より軽量なモデル（`base` や `tiny`）に書き換えて実行してください。
  ```python
  DEFAULT_MODEL_SIZE = "base"  # または "tiny" に下げる
  ```

## 3. 文字起こしの精度・挙動に関する不具合

### 🚨 症状：特定の固有名詞や専門用語がどうしても誤認識される
- **原因**: Whisper AIがその単語を一般的な別の言葉として解釈しています。
- **対処法**:
  - プロジェクトのルートフォルダにある `dictionary.txt` を開き、正しく認識させたい単語を1行に1つずつ正確に記述してください（例: `おたつ`, `BudouX` など）。

### 🚨 症状：話していない無音部分で、英語の字幕や意味不明な記号が連続して出力される
- **原因**: 音声内のノイズやBGMを、AIが無理やり言葉として聞き取ろうとして暴走（ハルシネーション）しています。
- **対処法**:
  - スクリプト内の設定で `language="ja"`（日本語固定）を指定しているため言語の誤判定は防げていますが、重度なノイズがある場合は、事前に動画編集ソフト側でBGMの音量を下げるか、ノイズ除去フィルターをかけた音声を切り出して入力してください。

---

## 4. ファイルの書き込みエラー

### 🚨 症状：「SRTファイルの書き込みに失敗しました」と表示される
- **原因**: 出力先となる `.srt` ファイルが、Premiere Pro やメモ帳などの別のソフトで開かれたまま（ロックされた状態）になっています。
- **対処法**:
  - 字幕ファイルを開いている他のアプリケーションをすべて完全に閉じてから、再度スクリプトを実行してください。
