# uv コマンド基本マニュアル

本プロジェクトでは、高速な Python パッケージ管理ツールである `uv` を使用しています。
`uv` のインストール方法から、日常的な開発・実行で使うコマンドまでのまとめです。

---

## 1. uv のインストール方法（初回のみ）

Windows 環境に `uv` を導入するには、ターミナル（PowerShell）を開いて以下のコマンドを実行してください。

```powershell
# Windows標準のwingetを使ってuvをインストールするコマンド
winget install Astral.uviex"
```

※インストール完了後、コマンドを有効化するために**一度ターミナルを閉じて、新しく開き直して**ください。
※ `uv --version` と叩いてバージョンが表示されれば導入成功です。

---

## 2. 最もよく使う基本コマンド（これだけで9割足ります）

### 🔧 1. 環境の同期（ライブラリの一括インストール）
`pyproject.toml` に書かれたライブラリの構成を読み込み、実際の仮想環境（`.venv`）に完全に一致させます。
プロジェクトを新しくダウンロードした直後は、まずこのコマンドを1回実行します。
```powershell
uv sync
```
* ※ 手動で仮想環境を作成したり、有効化（Activate）したりする操作は一切不要です。すべて自動で行われます。

### 🚀 2. スクリプトの実行
仮想環境の中に入る面倒な手順を踏むことなく、プロジェクトに登録されたコマンドを使って安全にスクリプトを一発実行します。
```powershell
uv run whisper-srt
```
* ※ 引数を渡して実行する場合の例：
  `uv run whisper-srt ./data/input.mp4 -m small`

## 3. 知っておくと便利なコマンド

### ➕ 1. ライブラリの新しく追加
プログラム内で新しく使いたい外部ライブラリができたときに実行します。
```powershell
uv add ライブラリ名
```
* 例: `uv add requests`
* ※ これを実行すると、`pyproject.toml` の更新、`uv.lock` の同期、実際のインストールまでが自動で一撃で行われます。

### ❌ 2. 不要になったライブラリの削除
使わなくなったライブラリをプロジェクトから綺麗に消去します。
```powershell
uv remove ライブラリ名
```

### 🧹 3. キャッシュやゴミファイルの掃除
内部の古いビルドキャッシュなどを一括削除し、環境をクリーンにリセットします。プログラムの挙動がおかしいと感じたときに有効です。
```powershell
uv clean
```

## 4. 注意すべき管理ファイルについて

- **pyproject.toml**（変更してOK）
  プロジェクトの設計図です。必要なライブラリを追加したい場合は、ここの `dependencies` に名前を追記します（または `uv add` を使います）。
- **uv.lock**（手動編集は絶対NG / Git管理は必須）
  寸分の狂いもない環境を再現するためのロックファイルです。`uv` が裏で自動管理するため、人間が手動で書き換えてはいけません。ただし、チームや未来の自分と環境を共有するために、Git のコミットには必ず含めてください。
- **.venv フォルダ**（手動編集は絶対NG / Git管理もNG）
  実際のライブラリの実体がダウンロードされるフォルダです。容量が巨大なため、`.gitignore` で Git の管理対象から除外します。
