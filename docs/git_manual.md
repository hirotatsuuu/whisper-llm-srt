# Git 基本操作マニュアル

本プロジェクトを Git および GitHub で管理・バックアップするための基本的なコマンドとワークフローのまとめです。

---

## 1. 基本的な開発ワークフロー（日常的に使う手順）

プログラムを変更したり、設定を書き換えたりした後は、以下の3ステップを1セットとして実行します。

### 📥 ステップ1. 変更したファイルをステージング（登録）する
新しく作ったファイルや、書き換えたファイルを変更予定リストに追加します。
```powershell
# すべての変更（新規作成・修正・削除）をまとめて登録する場合（推奨）
git add .

# 特定のファイルだけを指定して登録する場合
git add src/main.py
```

### 💾 ステップ2. コミット（変更の記録）
登録した変更に対して、「何を変えたのか」が分かるメッセージを添えてPC内に保存します。
```powershell
git commit -m "feat: スクリプトに新しい機能を追加"
```
* ※ メッセージの先頭に `feat:`（機能追加）、`fix:`（バグ修正）、`docs:`（説明書更新）などをつけると、後から履歴が見やすくなります。

### 🚀 ステップ3. GitHub へのプッシュ（アップロード）
PC内の変更履歴を、インターネット上の GitHub リポジトリへ同期します。
```powershell
git push
```

---

## 2. よく使う確認コマンド

### 📋 現在の状態を確認する
どのファイルが変更されているか、`.gitignore` で正しく無視されているかなどを確認します。
```powershell
git status
```

### 📜 過去の変更履歴を見る
これまでに誰がどんなコミットをしたのかを一覧で確認します。
```powershell
git log --oneline
```
* ※ キーボードの `q` キーを押すと、ログ画面から通常のターミナルに戻れます。

## 3. トラブルシューティング（もしもの時のコマンド）

### 🚨 症状：間違えて大容量の動画ファイルを `git add` してしまった
`.gitignore` を設定し忘れるなどして、数億バイトある動画ファイル（`.mp4`）や仮想環境（`.venv`）をステージングに登録してしまった場合の取り消し方法です。

**対処法**:
コミットする前（`git push` 前）であれば、以下のコマンドでステージングの登録だけを安全に解除できます。（※PC内の実際のファイルは消えません）
```powershell
# すべての登録をいったん白紙に戻す
git reset

# 特定のファイルだけ登録を解除する
git rm --cached data/large_movie.mp4
```

### 🚨 症状：改行コードの警告が出る
`git add` をした際に、`warning: LF will be replaced by CRLF` という警告が出ることがあります。
- **原因**: Windows と Mac で改行の扱いが異なるための自動警告です。
- **対処法**:
  本プロジェクトには すでに `.gitattributes` が配置されているため、**この警告は完全に無視してそのまま進めて問題ありません。** Gitが裏側で自動的に最適な改行コード（LF）に統一して保存してくれます。

## 4. このプロジェクトでGit管理に含めるもの・外すもの

本プロジェクトの `.gitignore` と `.gitattributes` のルールに基づいた適切な管理対象です。

| ファイル/フォルダ | Git管理の対象 | 理由 |
| :--- | :--- | :--- |
| `src/main.py` | ⭕ **含める** | プログラムの本体コードであるため。 |
| `pyproject.toml` / `uv.lock` | ⭕ **含める** | 他の環境で全く同じライブラリを再現するために必須なため。 |
| `README.md` / `docs/` | ⭕ **含める** | プロジェクトの使い方の取扱説明書であるため。 |
| `.gitignore` / `.gitattributes` | ⭕ **含める** | Gitの管理ルールそのものを共有するため。 |
| `.venv/` | ❌ **絶対に除外** | 自動生成される巨大なフォルダであり、`uv sync` でいつでも復元できるため。 |
| `data/` 内の動画・音声 | ❌ **原則除外** | 容量が大きく、個人の素材データであるため（テスト用の数秒の軽量ファイルを除く）。 |
